## How Can I Contribute?

### Reporting Bugs
- Use the issue tracker to report bugs
- Describe the bug and include steps to reproduce
- Include screenshots if applicable
- Mention your environment (OS, browser, Node version)

### Suggesting Enhancements
- Use the issue tracker to suggest features
- Clearly describe the feature and its benefits
- Include mockups or examples if possible

### Pull Requests
1. Fork the repository
2. Create a new branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test your changes thoroughly
5. Commit your changes (`git commit -m 'Add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Development Guidelines

### Code Style
- Use consistent indentation (2 spaces)
- Follow existing code patterns
- Add comments for complex logic
- Keep functions small and focused

### Backend
- Follow REST API best practices
- Add proper error handling
- Document new endpoints
- Use meaningful variable names

### Frontend
- Follow React/Next.js best practices
- Keep components reusable
- Use Tailwind CSS classes
- Ensure responsive design
- Test on multiple devices

### Commit Messages
- Use clear and descriptive commit messages
- Start with a verb (Add, Fix, Update, Remove)
- Keep the first line under 50 characters
- Add detailed description if needed

Example:
```
Add search debounce to improve performance

- Implemented 300ms debounce on search input
- Prevents excessive API calls
- Improves user experience
```

## Testing
- Test your changes locally
- Ensure both backend and frontend work together
- Check responsive design on mobile
- Test search functionality
- Verify error states

## Questions?
Feel free to open an issue for any questions!
